# Maccy 调研报告

> 项目地址：https://github.com/p0deje/Maccy
> 18.8k+ stars | MIT 协议 | macOS 14+

## 1. 项目概述

Maccy 是一个轻量级 macOS 剪贴板历史管理器。核心理念：**Lightweight, Keyboard-first, Secure, Native UI, Open source**。

---

## 1.5 诞生背景与发展

### 创始人

Alex Rodionov（GitHub: [p0deje](https://github.com/p0deje)），同时也是 Selenium 项目的 Ruby 维护者。

### 起因

Alex 从 Linux 转到 macOS 后，找不到一个像 Linux 上的 **Parcellite** 那样免费、简洁的剪贴板管理器。市面替代品要么臃肿要么收费，于是决定自己做一个，同时借此学习 Swift 和 macOS 开发。

> **Parcellite** 是 Linux 上极简的 GTK+ 系统托盘剪贴板管理器（GPLv3），是 Lubuntu 等轻量级发行版的默认选择。Maccy 本质上是它的 macOS 精神继承者。

### 2.3 剪贴板监听机制

**核心文件：`Clipboard.swift`**

```
Timer (500ms 间隔) → 检查 NSPasteboard.general.changeCount
  → changeCount 变化 → 读取 pasteboardItems
  → 过滤忽略类型（密码/临时/自定义正则）
  → 过滤忽略 App（按 bundleIdentifier）
  → 合并多 items 为单条 HistoryItem
  → 触发 onNewCopy hooks
```

关键设计点：
- **轮询模式**：用 `Timer.scheduledTimer` 每 500ms 检查一次 `pasteboard.changeCount`，而非事件通知（macOS 没有原生的剪贴板变化通知）
- **支持的类型**：`.fileURL`, `.html`, `.png`, `.rtf`, `.string`, `.tiff`
- **忽略机制**：内置忽略 `autoGenerated`/`concealed`/`transient` 类型，用户可配置忽略 App 和正则
- **去重**：写入时标记 `.fromMaccy` 类型，避免自己写入触发重复

### 2.4 来源应用识别（完整链路）

Maccy 能显示每条剪贴板记录来自哪个应用（名称 + 图标），实现如下：

**Step 1: 记录来源 bundleIdentifier**（`Clipboard.swift`）

```swift
// 取当前前台应用作为复制来源（推断，非精确追踪）
private var sourceApp: NSRunningApplication? {
  NSWorkspace.shared.frontmostApplication
}

// 检测到剪贴板变化时记录
historyItem.application = sourceApp?.bundleIdentifier  // e.g. "com.google.Chrome"
```

局限性：后台应用写入剪贴板时，记录的来源可能是前台应用。但对绝大多数场景准确。

**Step 2: bundleIdentifier → 应用名称**（`HistoryItemDecorator.swift`）

```swift
var application: String? {
  if item.universalClipboard { return "iCloud" }
  guard let bundle = item.application,
    let url = NSWorkspace.shared.urlForApplication(withBundleIdentifier: bundle)
  else { return nil }
  // /Applications/Google Chrome.app → "Google Chrome"
  return url.deletingPathExtension().lastPathComponent
}
```

**Step 3: bundleIdentifier → 应用图标**（`ApplicationImage.swift`）

```swift
var nsImage: NSImage {
  if let appURL = NSWorkspace.shared.urlForApplication(
    withBundleIdentifier: bundleIdentifier
  ) {
    return NSWorkspace.shared.icon(forFile: appURL.path)
  }
  return Self.fallbackImage  // "questionmark.app.dashed" SF Symbol
}
```

额外细节：用 `DispatchSource.makeFileSystemObjectSource` 监听应用文件变化——应用被删除时清缓存，应用被更新时重新获取图标。

**Step 4: 图标缓存**（`ApplicationImageCache.swift`）

用字典按 `bundleIdentifier` 缓存 `ApplicationImage` 实例，避免重复查询。

**链路总结：**

```
复制事件 → NSWorkspace.shared.frontmostApplication?.bundleIdentifier → 存入 HistoryItem.application
显示时   → NSWorkspace.urlForApplication(bundleId) → 应用路径
         → .deletingPathExtension().lastPathComponent → 应用名
         → NSWorkspace.icon(forFile:) → 应用图标
```

关键 macOS API：
- `NSPasteboard.general.changeCount` — 检测剪贴板变化
- `NSWorkspace.shared.frontmostApplication` — 获取前台应用
- `NSWorkspace.shared.urlForApplication(withBundleIdentifier:)` — bundle ID → 应用路径
- `NSWorkspace.shared.icon(forFile:)` — 应用路径 → 图标

### 2.4 数据持久化

- 使用 **SwiftData**（`@Model` + `ModelContainer`）
- SQLite 存储在 `~/Library/Application Support/Maccy/Storage.sqlite`
- `HistoryItem` 一对多 `HistoryItemContent`
- 每条 content 存储 `type: String`（如 `public.utf8-plain-text`）+ `value: Data?`（原始二进制）
- 这意味着剪贴板的**所有格式都完整保存**（string、html、rtf、图片等）

### 2.5 菜单栏实现

**不是用 `MenuBarExtra`，而是 `NSStatusItem` + `NSPanel`：**

```swift
// MaccyApp.swift - 用空 MenuBarExtra 避免 dock 图标
@main
struct MaccyApp: App {
  @NSApplicationDelegateAdaptor(AppDelegate.self) var appDelegate
  @State private var hiddenMenu: Bool = false
  var body: some Scene {
    MenuBarExtra("", isInserted: $hiddenMenu) { EmptyView() }
  }
}

// AppDelegate.swift - 真正的菜单栏图标
lazy var statusItem: NSStatusItem = {
  let statusItem = NSStatusBar.system.statusItem(withLength: .variableLength)
  statusItem.button?.action = #selector(performStatusItemClick)
  statusItem.button?.image = Defaults[.menuIcon].image
  return statusItem
}()
```

**弹出面板用自定义 `NSPanel`：**
- `FloatingPanel` 继承 `NSPanel`，style 包含 `.nonactivatingPanel`（不抢焦点）
- `level = .statusBar`（在最上层）
- 点击外部自动关闭（`resignKey` 时 close）
- 支持拖动和调整大小
- SwiftUI 内容通过 `NSHostingView` 嵌入

### 2.6 快捷键设计

- **全局快捷键**：通过 `KeyboardShortcuts` 库注册（默认 `⌘+Shift+C`）
- **两种模式**：
  - **Toggle 模式**：按一下打开，再按关闭
  - **Cycle 模式**：按住修饰键 + 连续按主键，在历史记录间循环，松开修饰键确认选择
- **项目快捷键**：前 9 个未置顶项自动分配 `⌘+1` ~ `⌘+9`
- **修饰键操作**：
  - 无修饰键 = 默认操作（粘贴/复制取决于设置）
  - `⌘` = 复制到剪贴板
  - `⌥` = 粘贴
  - `⌥+⇧` = 去格式粘贴

## 4. 对 Pasfo 的参考价值

### 4.1 直接可参考

| 参考点 | Maccy 实现 | Pasfo 如何借鉴 |
|--------|-----------|----------------------|
| **菜单栏 + 弹出面板** | `NSStatusItem` + 自定义 `NSPanel`（`FloatingPanel`） | 直接复用这个模式：菜单栏图标 + 弹出面板显示检测结果和转换按钮 |
| **剪贴板监听** | `Timer` 轮询 `NSPasteboard.general.changeCount` | 同样方式监听，但不需要存历史，只需要检测当前剪贴板内容格式 |
| **剪贴板写回** | `pasteboard.clearContents()` + `setData/setString` 按类型写入 | 转换后用同样 API 写回，关键是写入 `.html` 类型让 Apple Notes 识别为富文本 |
| **快捷键** | `KeyboardShortcuts` 库 | 注册全局快捷键（如 `⌘+Shift+V`）触发格式转换 |
| **App 入口模式** | `MenuBarExtra(hidden)` + `AppDelegate` | 相同模式：无 Dock 图标，只有菜单栏 |
| **偏好设置** | [Settings](https://github.com/sindresorhus/Settings) 库 | 轻量设置面板（默认转换目标等） |

### 4.2 关键技术细节

#### 写入富文本到剪贴板（Apple Notes 可识别）

Maccy 的 `Clipboard.copy()` 方法展示了如何**按类型写入多种格式**：

```swift
pasteboard.clearContents()
for content in contents {
  pasteboard.setData(content.value, forType: NSPasteboard.PasteboardType(content.type))
}
```

Pasfo 需要做的是：
1. 读取 `.string` 类型的纯文本
2. 检测格式（box-drawing / markdown / 代码）
3. 转换为 HTML
4. **同时写入 `.html` 和 `.string` 两种类型**，这样：
   - Apple Notes 粘贴时读取 HTML → 显示为富文本表格
   - 纯文本编辑器粘贴时读取 string → 显示为纯文本

```swift
// Pasfo 核心写入逻辑（伪代码）
pasteboard.clearContents()
pasteboard.setData(htmlData, forType: .html)       // Apple Notes 会读这个
pasteboard.setString(markdownText, forType: .string) // 文本编辑器读这个
```

#### Apple Notes 特有类型

Maccy 定义了 `com.apple.notes.richtext` 类型（`NSPasteboard.PasteboardType+Types.swift`），说明 Apple Notes 有专有的 pasteboard 类型，可能需要额外处理。

#### NSPanel 不抢焦点

`FloatingPanel` 使用 `.nonActivatingPanel` style，这对 Pasfo 很重要——转换面板不应该抢走当前 App 的焦点。

### 4.3 不需要参考的部分

- **SwiftData 持久化**：Pasfo 不需要存历史，只做实时转换
- **搜索/模糊匹配**：不需要
- **Pin/排序/PasteStack**：不需要
- **多 pasteboardItem 合并**：不需要

## 5. 总结

Maccy 对 Pasfo 最大的参考价值是：

1. **菜单栏 App 的完整模板**——`MaccyApp` + `AppDelegate` + `FloatingPanel` 这套组合可以直接复用
2. **剪贴板读写的最佳实践**——轮询检测 + 多类型同时写入
3. **第三方库选型**——`KeyboardShortcuts` + `Defaults` 是 macOS 独立 App 的成熟选择
4. **NSPanel 浮动面板**——不抢焦点的弹出面板实现

Pasfo 本质上比 Maccy 简单很多（不需要历史、搜索、持久化），可以在 Maccy 的架构骨架上做减法。
